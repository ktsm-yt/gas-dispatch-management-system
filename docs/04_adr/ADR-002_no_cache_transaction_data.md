# ADR-002: 当日トランザクション非キャッシュ + 更新検知

## ステータス

承認済み

## コンテキスト

ダッシュボードは常に開きっぱなしで運用される想定があり、以下の課題がある：

1. 他ユーザーの更新が画面に反映されない（"画面変わらねぇ"問題）
2. キャッシュによる古いデータの表示リスク
3. パフォーマンスとリアルタイム性のトレードオフ

## 決定

**当日のトランザクションデータ（案件/配置）は原則キャッシュしない。更新検知用のメタAPI（`getDashboardMeta`）でポーリングする。**

## 理由

- **データ整合性優先**: 配車業務では古いデータによるミスが致命的
- **シンプルな実装**: キャッシュ無効化ロジックの複雑化を回避
- **更新検知の分離**: 全データ再取得ではなく、メタ情報（`max(updated_at)`）だけをチェック

## 詳細

### キャッシュ方針

| データ種別 | キャッシュ | TTL | 備考 |
| --- | --- | --- | --- |
| マスター（顧客/スタッフ） | する | 60〜300秒 | 更新時に破棄 |
| トランザクション（案件/配置） | しない | - | 常に最新を取得 |

### 更新検知フロー

1. フロントが`getDashboardMeta(date)`を定期呼び出し（例: 30秒間隔）
2. 返却される`maxUpdatedAt`を前回と比較
3. 差分があればトースト表示「更新があります」→ ユーザー操作で再読込

- [06_backend.md](../03_spec/06_backend.md) - `getDashboardMeta` 仕様・キャッシュ方針

## 影響

- Spreadsheetへのアクセス頻度が増加（許容範囲内）
- ユーザーは明示的に再読込を行う（自動更新ではない）
- 将来的にWebSocketへの移行も選択肢として残る
