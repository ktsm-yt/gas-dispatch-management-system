# 建設労務派遣業 案件配置管理システム — システム概要

## プロジェクト概要

**業界**: 建設労務派遣業（日雇い労務者の配置管理）

**解決した課題**:
毎日の案件配置・請求書作成・給与計算をExcelと手作業で管理していたため、転記ミス・集計漏れ・残業対応が常態化していた。

**成果**:

- 請求書一括処理: **手作業6時間 → 集計2分+帳票出力30分〜**（PDF/Excel選択出力、100社一括対応）
- 配置ミスゼロ（楽観的ロック + 差分保存で二重書き込みを防止）
- 案件・スタッフ・請求・支払の一元管理（モジュール間データ連携）

**開発体制**: ソロ開発

- PM / PO / UX設計 = 自分
- Developer = Claude Code
- Reviewer = Codex CLI
- Scribe（議事・仕様整理）= Gemini

## GASプラットフォーム制約と対策

GASは「追加インフラ不要」という最大のメリットがある一方、通常のWebバックエンドにはない制約が多い。
以下はその制約を実務レベルで克服した対策の一覧。

| 制約                                    | 影響                    | 対策                                                          |
| --------------------------------------- | ----------------------- | ------------------------------------------------------------- |
| 実行時間6分制限                         | バッチ処理が中断する    | `PropertiesService`で進捗保存、フロントから連続呼び出し     |
| シートI/O遅延（100ms〜数秒/回）         | 画面表示が遅い          | バルクI/O（`getValues`/`setValues`）、SWR的キャッシュ戦略 |
| 並列実行不可                            | 一括処理が遅い          | 10件バッチ＋自動再開、プログレスバー表示                      |
| HtmlService制約                         | SPA構築不可、iframe制限 | Vanilla JS/CSSでSPA級UX、z-indexモーダル管理                  |
| E2Eテスト不可                           | OAuth認証がBOT検知      | 3層代替戦略（ユニットテスト＋AIチート検出＋Chrome手動検証）   |
| スプレッドシート行数上限（〜100,000行） | データ肥大化            | 年度分割アーカイブ（年次自動移行）                            |
| PageBreak APIなし                       | 帳票レイアウト制御不可  | 中間スプレッドシート＋シート合体＋PDF変換                     |
