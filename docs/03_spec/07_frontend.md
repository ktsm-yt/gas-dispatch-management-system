## 7.3 ダッシュボード仕様

会社の肝となるメイン画面。6列カンバン形式で当日の案件を一覧表示。

### 列構成

| 列 | 時間区分 | 表示内容 |
|----|----------|----------|
| 1 | 上棟 | 棟上げ作業案件 |
| 2 | 終日 | 終日作業案件 |
| 3 | AM | 午前作業案件 |
| 4 | PM | 午後作業案件 |
| 5 | 夜勤 | 夜間作業案件 |
| 6 | 開始時間未定 | 時間未確定案件 |

### 列ヘッダー表示

```
+--------------------------------------------------+
| 2025年12月15日(月)  総件数:43  配置済:41  未配置:2 |
+--------------------------------------------------+
| 上棟    | 終日    | AM      | PM      | 夜勤    | 未定    |
|   10   |    3   |   22   |    6   |    0   |    2   |
|  -2    |   +0   |   +1   |   -1   |   +0   |   +0   |  <- 過不足
+--------+--------+--------+--------+--------+--------+
```

### 案件カードUI

```
+----------------------------------+
| [鳶] [揚げ]        ○○建設       |  <- タグ + 顧客名
+----------------------------------+
| 現場: △△マンション新築           |  <- 現場名
| 高橋, 田中, 佐藤                 |  <- 配置スタッフ
|                            3名   |  <- 人数バッジ
+----------------------------------+
```

### インタラクション

| 操作 | 動作 |
|------|------|
| ホバー | 案件詳細のプレビューツールチップ（閲覧のみ） |
| クリック | 編集モーダル表示（スタッフ配置・案件編集） |
| 日付左右矢印 | 前日・翌日に切り替え |

### 開きっぱなし対策（更新検知）

毎日ハードに使う運用を想定し、以下を推奨。

- 画面上部に「最終更新（取得）時刻」を表示
- 一定間隔で当日データの更新有無（例：`getDashboardMeta(date)`）を確認し、差分があれば「更新があります」トースト＋更新ボタンで再読込
- 保存直後は該当カードのみ更新（全件再読込しない）

## 7.4 案件編集モーダル

```
+------------------------------------------+
|  案件編集                           [X]  |
+------------------------------------------+
| 顧客:     [○○建設         ▼]           |
| 現場名:   [△△マンション新築工事    ]    |
| 現場住所: [東京都...              ]      |
| 作業日:   [2025-12-15]                   |
| 時間区分: [AM ▼]                         |
| 開始時間: [08:00 ▼]（任意）              |
| 必要人数: [3]                            |
| 作業種別: [x]鳶 [x]揚げ [ ]鳶揚げ        |
+------------------------------------------+
| 請求書用項目（書式により表示）            |
| 発注ナンバー: [033930            ]       |
| 営業所:       [特需2             ]       |
| 担当工事課:   [○○工事課          ]       |
| 担当監督名:   [山田 太郎          ]       |
| 物件コード:   [A-12345           ]       |
+------------------------------------------+
| スタッフ配置:                            |
| +--------------------------------------+ |
| | 高橋太郎  表示:AM 給与:終日 請求:終日  ¥XX,XXX  23区内 ¥XXX | |
| | 田中次郎  表示:AM 給与:AM   請求:AM    ¥XX,XXX  23区外 ¥XXX | |
| | [+ スタッフ追加]                     | |
| +--------------------------------------+ |
+------------------------------------------+
|           [キャンセル]  [保存]           |
+------------------------------------------+
```

- 「請求書用項目」は顧客の書式（`invoice_format`）に応じて表示/必須を切り替える（例：様式2=発注No/営業所（帳票は別列出力）、様式3=担当工事課/担当監督名/物件コード）
- 配置行は「表示（時間区分）」と「給与/請求の区分」を分離して保持し、**表示=AMだが支給=終日**のような現場運用を吸収する
- 交通費：エリア選択でマスター金額を自動セット（上書き時は手入力フラグONで保存）

## 7.5 スタッフ選択（インクリメンタルサーチ）

```
入力: [た          ]
      +------------------+
      | 高橋太郎  鳶     |
      | 田中次郎  揚げ   |
      | 田村三郎  鳶揚げ |
      +------------------+
```

- 1文字入力から絞り込み開始
- 氏名・カナで検索
- NG顧客の案件では該当スタッフを非表示
- 当日他案件配置済みスタッフには警告表示
- バイク有無（`has_motorbike`）は属性として表示可能（例: バイクタグ）。単価は自動変更しない

## 7.6 その他機能

### LINEテンプレート
配置情報からLINE送信用テキストを生成し、クリップボードへコピーできる形で提示（自動送信はしない）。

### 帳票出力
- **作業員名簿**: 全国統一様式5を想定（案件/配置情報から生成）
- **請求書（帳票）**: 顧客別・月別に集計し、PDF出力（必須）／Excel出力（追加費用で合意済）／編集して出力（Spreadsheet別タブ経由）に対応。顧客ごとの書式切替に対応
- **{{FORMAT3_TYPE}}専用**: 専用Excelフォーマットに合わせた出力（簡易印影対応）

> **用語の区別（揉め防止）**
> - **請求書（帳票）**: PDF/xlsx/編集して出力（請求書レイアウト）
> - **請求データ（明細/集計）**: 請求管理画面のExcel/CSVエクスポート（集計データ）

> **請求管理画面のボタン例**
> - 帳票: `[PDF出力] [Excel出力] [編集して出力]`
> - データ: `[請求データ（Excel/CSV）]`

### 未配置一覧
未配置案件を一覧し、スタッフを割り当てる。割当完了後はダッシュボードへ自動反映。

## 7.7 ダッシュボード実装方針

日々ハードに使われる前提のため、体感速度と「更新されている感」を最優先にする。

### (A) 取得は"当日分だけ"
- `getDashboard(date)` は対象日分だけ返す
- 必要フィールドを最小にし、詳細は `getJob(jobId)` で取得

### (B) 初期描画は"カード一覧だけ"、詳細は遅延ロード
- 初期描画は「カードを描画できる情報」に限定
- モーダルは「スケルトン→詳細反映」の2段階で体感を落とさない

### (C) 更新は"該当カードだけ差し替え"
- DOM要素に `data-job-id` を付け、該当カードのみ更新
- 列全体は再描画しない

### (D) 検索は debounce + 事前Index化
- インクリメンタルサーチは `debounce(150〜250ms)`
- 候補表示は上限20件

### (E) DOM操作はまとめて描画
- `DocumentFragment` / 一括 `innerHTML` / `<template>` を使用
- ループ内で `querySelector` や reflow を誘発しない

### (F) CSSはアニメーション控えめ
- 影・ぼかし・高頻度アニメーションは最小化
- トースト/モーダルは150ms、`transform/opacity` のみ
