# 11. デプロイ手順

## 11.1 環境構成

| 環境 | ブランチ | GASデプロイ | スプレッドシート | 使用者 |
|------|----------|-------------|------------------|--------|
| **本番（prod）** | main | バージョン固定 | 本番DB | 番頭さん、社長 |
| **開発（dev）** | feature/* | Head（最新） | 開発DB | 開発者 |

### GASでの環境分離

同一プロジェクトで複数デプロイを作成:

1. GASエディタ → 「デプロイ」→「デプロイを管理」
2. **本番用**: バージョン番号を固定（例: v5）
3. **開発用**: 「Head」を選択（常に最新コード反映）

## 11.2 本番デプロイチェックリスト

### 更新前

- [ ] 顧客からOKをもらった
- [ ] 開発環境での全機能テスト完了
- [ ] 本番DBのバックアップ取得（スプレッドシートをコピー）
- [ ] **番頭さんに更新予定を事前連絡し、了承を得た**
- [ ] 変更内容のドキュメント更新

### デプロイ手順

1. [ ] ENV変数をPRODに設定
2. [ ] `clasp push` 実行
3. [ ] GASエディタで動作確認
4. [ ] Webアプリとして新バージョンデプロイ
5. [ ] 本番環境での動作確認
6. [ ] クライアントへの展開通知

### 更新後

- [ ] 本番環境で基本動作確認
- [ ] 問題があれば即座に前バージョンに戻す準備ができている
- [ ] Gitタグを打った（`demo-YYYYMMDD-<topic>`）

## 11.3 デプロイ手順詳細

### 1. 環境変数の確認

```javascript
// GASエディタで実行
function checkEnv() {
  const env = PropertiesService
    .getScriptProperties()
    .getProperty('ENV');
  Logger.log('Current ENV: ' + env);
}
```

### 2. コードのプッシュ

```bash
# 差分確認
clasp status

# プッシュ
clasp push
```

### 3. Webアプリデプロイ

GASエディタで:
1. 「デプロイ」→「新しいデプロイ」
2. 種類: 「ウェブアプリ」
3. 説明: バージョン番号と変更内容
4. 実行ユーザー: 「ウェブアプリにアクセスしているユーザー」
5. アクセスできるユーザー: 「[ドメイン]内のすべてのユーザー」
6. 「デプロイ」

または:
```bash
clasp deploy --description "v1.0.0 - 初回リリース"
```

## 11.4 ロールバック手順

### 方法1: GASエディタから

1. 「デプロイ」→「デプロイを管理」
2. 前バージョンの「︙」→「アクティブなデプロイに変更」

### 方法2: claspから

```bash
# デプロイ一覧確認
clasp deployments

# 特定バージョンをアクティブに
clasp deploy --deploymentId [PREVIOUS_DEPLOYMENT_ID]
```

### DBロールバック

1. Google DriveでスプレッドシートのVersion履歴を開く
2. 該当バージョンを選択
3. 「このバージョンを復元」

## 11.5 リリースノート管理

### フォーマット

```markdown

### 新機能
- ダッシュボード6列表示対応

### 修正
- スタッフ検索の文字化け修正

- 時間区分に「開始時間未定」追加
```

### 命名規則

- メジャーバージョン: 大きな機能追加・破壊的変更
- マイナーバージョン: 機能追加・改善
- パッチバージョン: バグ修正

例: `v1.2.3`
- 1: メジャー
- 2: マイナー
- 3: パッチ

## 11.7 バックアップ/復旧

### 日次バックアップ（推奨）

当日分のスプレッドシートコピーをDriveに作成するスクリプトを用意。

```javascript
// 例: トリガーで毎日実行
function dailyBackup() {
  const ss = SpreadsheetApp.openById(MAIN_DB_ID);
  const folder = DriveApp.getFolderById(BACKUP_FOLDER_ID);
  const date = Utilities.formatDate(new Date(), 'Asia/Tokyo', 'yyyyMMdd');
  ss.copy(`backup_${date}`).moveTo(folder);
}
```

### 障害時復旧

1. バックアップからの復元（スプレッドシートコピー）
2. または操作ログ（T_AuditLog）に基づく差分復旧
