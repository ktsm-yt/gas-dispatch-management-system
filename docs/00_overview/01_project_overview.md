# 1. プロジェクト概要

## 1.1 システム概要

建築業界向け人材派遣業における、案件管理から請求・給与計算までの一連のバックオフィス業務を支援するWebアプリケーション。

**規模感**
- 1日の案件数: {{DAILY_JOB_COUNT_RANGE}}
- スタッフ数: {{STAFF_MASTER_COUNT}}
- 顧客数: {{CUSTOMER_MASTER_COUNT}}（既存データ）
- 月間売上: {{MONTHLY_REVENUE}}

## 1.2 開発フェーズ

| フェーズ | 内容 | 金額 |
|----------|------|------|
| Phase 1 | 基幹機能（案件・配置・顧客・スタッフ管理） | {{PHASE1_COST}} |
| Phase 2 | 請求機能（請求・給与・外注費・売上集計） | {{PHASE2_COST}} |
| 追加 | PDF出力機能（請求書・送り状・{{FORMAT3_TYPE}}専用） | {{ADD_PDF_COST}} |
| 追加 | Excel出力機能（請求書帳票） | 別途見積（合意済） |
| 追加 | 初期データ移行（顧客{{CUSTOMER_MASTER_COUNT}}・スタッフ{{STAFF_MASTER_COUNT}}） | 別途見積（合意済） |

## 1.3 主要ステークホルダー

| 役割 | 主な関心事 | 利用機能 |
|------|------------|----------|
| 社長 | 売上管理・請求確認・経営判断 | 売上分析・請求管理 |
| 番頭 | 日々の配置管理・現場運営 | ダッシュボード・配置管理・案件管理 |

## 1.4 システム導入目的

1. **属人化からの脱却** - 1人に集中している業務を複数人で運用・引き継ぎ可能に
2. **二重入力の排除** - 1回の入力で「受注→配置→請求→給与」に自動連動
3. **金銭管理の信頼性向上** - 入力ミス・請求漏れ・給与計算ミス・二重払いを削減
4. **分析基盤の構築** - 売上・稼働状況の可視化と経営判断支援

## 1.5 初期データ移行（合意事項）

- 移行対象（初期投入）: `M_Customers`（顧客 {{CUSTOMER_MASTER_COUNT}}）、`M_Staff`（スタッフ {{STAFF_MASTER_COUNT}}）
- 前提: 先方よりExcel/CSV等の元データが提供されること（列定義・表記ゆれはすり合わせ）
- 対象外（別途切り分け）: 顧客別単価の複雑な例外、交通費マスター、請求書の独自フォーマット等（必要なら箇条書きで追加合意）
